# Core 2-Step Fused Interior Result

Date: 2026-06-07

## Status

`CUDA3D_CORE_2STEP_FUSED_INTERIOR` stage 1-3 are complete:

- design document written;
- meaningful 1GPU case generated;
- debug-only fused p2 predictor implemented;
- p2-shift correctness passed on the meaningful case;
- receiver output stayed bitwise identical to `zmem_reference`.

This is not a speedup result yet. The current fused predictor is a correctness probe: it locally recomputes the first-step values needed by `p(t+2)` and writes only the auxiliary debug buffer. It does not replace baseline `p_core`, does not commit `p(t+2)`, and is not a performance path.

## Decision

Proceed to `CUDA3D_CORE_2STEP_FUSED_COMMIT` only after designing the real fused kernel:

- compute `p(t+1)` and strict-inner `p(t+2)` inside one fused core kernel;
- stage first-step data inside a CTA/local tile so `p(t+2)` does not read another CTA's unfinished `p(t+1)`;
- write `p(t+1)` to global `p0` for baseline-compatible source/receiver/PML ordering;
- commit/skip only validated inner tiles;
- use block-level early skip before any shared-memory fill for fully covered committed tiles.

Next acceptance gates:

```text
commit correctness: rel_l2 <= 1e-5, no NaN/Inf
meaningful repeat speedup: >= 5% to continue fused route
perf_1gpu_6shots repeat speedup: >= 2% to keep candidate
```
